## 3. 完善自动测试流程与CSV配置加载
- **UI 调整**：
  - 矢网仪器IP设置区恢复 `portLabel` + `portLineEdit`，默认值 `5025`。
  - 状态图标（`statusIconLabel` + `statusTextLabel`）已移至开始按钮旁（用户在 Qt Designer 中手动调整）。
- **CSV 配置加载** (`loadConfigCsv`)：
  - 解析 CSV 文件，第一行作为表头。
  - **隐藏第一列**（序号列），使用 `QTableWidget` 自带行号代替。
  - 内容填充到 `configTableWidget`，自动调整列宽。
- **自动测试流程状态机**：
  - 必须依次完成：`连接矢网` → `选择波位文件夹` → `选择配置文件` → `天线加电`，`开始自动测试` 按钮才可用。
  - 新增 `canStartTest()` 和 `updateStartButtonState()` 控制按钮使能状态。
  - `on_startAutoTestButton_clicked()` 会检查所有前置条件并给出具体提示。
- **天线加电/断电解耦设计**：
  - `AntennaDeviceWindow` 提取公共方法 `antennaPowerOn()` / `antennaPowerOff()`，内部实现协议发送；UI 槽函数 `on_antennaPowerOnButton_clicked()` / `on_antennaPowerOffButton_clicked()` 仅调用公共方法。
  - `AutoTestWindow` 通过信号 `antennaPowerOnRequested()` / `antennaPowerOffRequested()` 发出请求，不直接依赖 `AntennaDeviceWindow`。
  - `MainWindow` 在创建 `AutoTestWindow` 时通过信号槽连接：`antennaPowerOnRequested → antennaPowerOn`，`antennaPowerOffRequested → antennaPowerOff`。
  - 优势：`AutoTestWindow` 与 `AntennaDeviceWindow` 完全解耦，由 `MainWindow` 统一协调，便于后续扩展（如日志记录、权限校验、批量控制等）。
- **矢网仪器 TCP 连接解耦（独立 socket）**：
  - `MainWindow` 新增独立成员 `m_vnaSocket` / `m_vnaConnected`，与主页面客户端模式的 `m_clientSocket` 完全分离。
  - `AutoTestWindow` 发射 `connectToHostRequested(ip, port)` / `disconnectFromHostRequested()` 信号，由 `MainWindow` 通过 Lambda 槽函数创建独立的 `m_vnaSocket`、执行 `connectToHost()`。
  - 连接成功/失败/断开的回调通过 `onConnected()` / `onDisconnected()` 公共槽传回 `AutoTestWindow`，更新其 UI 状态。
  - 新增 `onVnaSocketReadyRead()` / `onVnaSocketDisconnected()` 槽函数处理矢网数据接收和断线通知。
  - 主页面客户端模式（`onClientSocketDisconnected`）不再干扰矢网连接状态。
- **架构分析与重构建议**：生成 `CODE_REVIEW_ARCHITECTURE.md`，梳理了 P0/P1/P2 三级问题清单，提出了四层架构（UI → 业务 → 服务 → 数据）和分四阶段重构路线图。
