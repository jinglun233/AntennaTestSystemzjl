2. **修复 CustomTabWidget 分离窗口尺寸问题（第三次迭代，最终修复）**：
   - **问题**：`sizeHint()` 返回的是布局最小需求尺寸，不是 UI 设计器画布的原始几何尺寸
   - **现象**：弹出的分离窗口被截断，表格只显示部分行/列（如天线设备窗口设计器 1173×807，弹出窗口远小于此）
   - **根因**：widget 被 addTab() 后布局接管，原始几何尺寸丢失；sizeHint() 不等于设计器尺寸
   - **最终方案**：
     - CustomTabWidget 新增 `recordDesignSize(QWidget*, QSize)` 公开方法 + `m_designSizes` 成员映射
     - MainWindow 中在每个 widget 的 `addTab()` **之前**调用 `recordDesignSize(widget, widget->size())`
     - `handleTabDetached()` 优先从 `m_designSizes` 取设计尺寸，fallback 到当前尺寸
     - 屏幕居中使用 `QScreen::availableGeometry()` 计算
   - **提交**：`4cff34c` fix: 分离窗口使用UI设计器原始几何尺寸（替代不可靠的sizeHint）
   - **经验教训**：Qt 中获取 UI 设计器原始尺寸必须在 addTab/setParent 之前捕获；sizeHint() 不可靠
